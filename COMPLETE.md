## 💡 Design Decisions Explained

### Why Not a Dashboard?
- Dashboards feel overwhelming for busy students
- Too many controls and options
- Doesn't feel like a product experience
- Creates decision fatigue

### Why a 3-Step Generator?
- Progressive disclosure reduces cognitive load
- One focus at a time (budget → preferences → ingredients)
- Feels like a guided journey
- Matches user mental model

### Why Red & Gold?
- Red = Action, energy, food
- Gold = Warmth, quality, premium feel
- Together = Energetic but approachable
- Perfect for food/recipe context

### Why Spring Easing?
- More playful than standard easing
- Feels responsive and alive
- Not too bouncy, still professional
- Matches "friendly" brand personality

### Why Staggered Animations?
- Prevents visual overload
- Guides eye through content
- Feels more polished
- Creates sense of progression

## 🔧 Easy Customization

### Change Colors
Edit `:root` variables in `styles.css` lines 5-34:
```css
--primary-red: #FF5A52;
--primary-navy: #1a1a3e;
--primary-gold: #FFC846;
/* etc... */
```

### Add Recipes
Edit `recipeDatabase` in `script.js` lines 26-65:
```javascript
{
    id: 13,
    name: 'Your Recipe',
    emoji: '🍲',
    price: 3.50,
    time: 25,
    // etc...
}
```

### Modify Animations
Edit keyframes in `styles.css` lines 824-897:
```css
@keyframes yourAnimation {
    from { /* start state */ }
    to { /* end state */ }
}
```
